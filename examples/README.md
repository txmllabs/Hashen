# Example artifacts and fixtures

Deterministic sample input and expected output **structures** for Hashen. No secrets; artifact is small and synthetic.

## Contents

- **sample_artifact.bin** – Small deterministic artifact (generated by `generate_fixtures.py` so it matches the seal).
- **expected_seal_structure.json** – Example seal record (EPW) for the sample artifact and fixed config. Run `generate_fixtures.py` to regenerate; same artifact + config → same seal.
- **expected_manifest_structure.json** – Example bundle manifest structure (file list, seal_hash, audit_head_hash).
- **sample_verify_result.json** – Example verify output (ok, reason, audit_head_hash, seal_hash).
- **cache_demo.py** – Demo script: run pipeline twice on same input, report cache hit and timing.

## Regenerating fixtures

From repo root:

```bash
python examples/generate_fixtures.py
```

This writes `sample_artifact.bin` and the expected JSON files. The seal hash and payload are deterministic for the artifact bytes and config in the script.

## Reproducing verification

```bash
hashen-bundle examples/sample_artifact.bin example_run --output-dir /tmp/bundle
hashen-verify /tmp/bundle
```

The seal in the bundle should match `expected_seal_structure.json` (same config and artifact).

## Large binaries and secrets

Do not add large binaries or secrets to this directory. Fixtures are small and synthetic only.
